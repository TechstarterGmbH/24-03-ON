# Sprint Planung

- Der Scrum Master organisiert und moderiert das Sprint Planning Meeting (Und alle weiteren Scrum Meetings)
- Geht durch euren Backlog (eure Issues auf Github von gestern)
- Wählt die Issues aus, die ihr in den nächsten Woche umsetzen wollt (Product Owner muss die Prioritäten festlegen)
- Schätzt den Arbeitsaufwand für jedes Issue
- Schreibt die geschätzten Stunden in die Issues
- Weißt die Issues den Teammitgliedern zu
- Es können auch Issues in mehrere Teile aufgeteilt werden
- Mehrere Teammitglieder können an einem Issue arbeiten
- Jedes Teammitglied sollte an mindestens einem Issue arbeiten
- Jedes Issue sollte von mindestens einem Teammitglied bearbeitet werden

## Sprint Backlog

- Der Sprint Backlog ist die Liste der Issues, die ihr in den nächsten Woche umsetzen wollt
- Der Sprint Backlog sollte realistisch sein und alle Issues sollten in der gegebenen Zeit umsetzbar sein

## Let's go!

- Startet mit der Umsetzung eurer Issues
